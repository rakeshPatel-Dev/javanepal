# PWA (Progressive Web App)

JavaNepal is a fully installable PWA. Users can add it to their home screen on desktop and mobile for an app-like experience with offline support.

## Manifest

The Web App Manifest is at `/favicon/site.webmanifest`. It includes:

| Field | Value |
|-------|-------|
| `name` | JavaNepal — Master OOPs With Java |
| `short_name` | JavaNepal |
| `display` | `standalone` (no browser chrome) |
| `display_override` | `window-controls-overlay` (desktop PWA) |
| `theme_color` | `#0037b0` (primary blue) |
| `background_color` | `#ffffff` |
| Icons | 192x192, 512x512 (maskable), 96x96, SVG |
| Shortcuts | Units, Search, Bookmarks |

## Service Worker

Generated by [`@serwist/next`](https://serwist.pages.dev/) during production build.

### Source

`app/sw.ts` — used only by Serwist; not a route. Written in the Serwist API:

```ts
import { defaultCache } from "@serwist/next/worker"
import { installSerwist } from "@serwist/sw"

installSerwist({
  precacheEntries: (self as any).__SW_MANIFEST,
  skipWaiting: true,
  clientsClaim: true,
  navigationPreload: true,
  runtimeCaching: defaultCache,
})
```

### Build Output

`public/sw.js` is generated by `next build --webpack`. It is **not** tracked in git — Vercel generates it during deploy.

### What the Service Worker Does

1. **Precaches** all JS chunks, CSS, fonts, images, icons at install time
2. **Runtime caching** strategies for different asset types:
   - `CacheFirst` — static JS, CSS, fonts, images
   - `StaleWhileRevalidate` — pages, Next.js data, API responses
   - `NetworkFirst` — cross-origin requests
   - `NetworkOnly` — auth endpoints
3. **Navigation preload** — speeds up page navigations
4. **Auto-update** — `skipWaiting` + `clientsClaim` so new SW versions take control immediately
5. **Offline fallback** — users see cached pages when offline

## Build Command

The `package.json` build script uses `--webpack` flag because `@serwist/next` injects a webpack plugin:

```json
"build": "next build --webpack"
```

This is required for the service worker to be bundled. In `next dev` (Turbopack), the service worker is not generated.

## Installation Prompt

Browsers automatically prompt installation when they detect:
- A valid `site.webmanifest` with `display: standalone`
- A registered service worker with a `fetch` event handler
- The site is served over HTTPS (Vercel provides this)

Common triggers for the prompt:
- Visiting the page multiple times within a session
- Spending ~30 seconds on the page
- Certain user interactions (click, scroll)

## Verification

1. Deploy to Vercel
2. Open Chrome DevTools → Application → Manifest — verify manifest loads
3. Application → Service Workers — verify SW is registered and activated
4. Lighthouse → PWA audit — should pass all installable criteria
5. Click the install icon in the address bar, or use DevTools → Application → Service Workers → "Refresh" to test offline
